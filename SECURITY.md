# Security Policy

## Supported Versions

Until a production release is declared, only the default branch is supported for security fixes.

## Reporting a Vulnerability

Do not open public issues for suspected vulnerabilities, leaked credentials, or production security defects.

Report privately through GitHub private vulnerability reporting if it is enabled for this repository, or contact the repository owner directly through a trusted private channel. Include the affected repository, commit or version, reproduction steps, security impact, and logs or screenshots with secrets removed.

## Handling

- Acknowledge credible reports within 3 business days.
- Triage severity and exposure before merging remediation.
- Rotate any exposed credentials before or during the fix.
- Keep remediation details private until a fix is available.
